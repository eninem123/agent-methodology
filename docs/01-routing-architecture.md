# 任务路由架构演进

## 旧思路：全靠 RAG

```
用户问题
    ↓
向量检索
    ↓
拼进 prompt
    ↓
生成答案
```

**问题**：所有问题都走同一条路径，效率低、成本高、幻觉风险大。

---

## 新思路：分场景路由

```
用户问题
    ↓
┌─────────────────────────────────────────┐
│           分场景路由                    │
├─────────────┬─────────────┬─────────────┤
│ 不确定决策  │ 改表口径    │ 查生产      │
│             │             │             │
│ Bayesian    │ 读当日      │ MCP 查数    │
│ Skill +     │ model_csv / │ → Agent     │
│ 证据        │ 守卫        │ 生成 + Hook │
│             │             │ 拦截        │
├─────────────┼─────────────┼─────────────┤
│ 历史坑      │ 对话问答    │             │
│             │             │             │
│ TF-IDF      │ RAG         │             │
│ 教训库      │ 可选        │             │
└─────────────┴─────────────┴─────────────┘
```

**优势**：
- 不同场景用最适合的工具
- 减少不必要的 RAG 调用
- 成本可控、幻觉风险低
- 可扩展、可维护

---

## 任务路由决策矩阵

| 任务类型 | 最佳工具 | 输入 | 输出 |
|----------|----------|------|------|
| 不确定决策（选型/上线） | Bayesian Skill | 假设 + 证据分级 | 决策报告 |
| 改表/口径确认 | 门卫 + 当日设计稿 | 表名 + model_csv | 字段清单 + 冲突提示 |
| 查生产数据 | MCP + Agent | SQL + 目的说明 | 摘要 + 关键行 |
| 历史教训查询 | TF-IDF 知识库 | 问题关键词 | 相关教训 |
| 日常对话问答 | RAG（可选） | 自然语言问题 | 自然语言答案 |
